# Medical-AI-Core

This project is a preprocessing and training pipeline that prepares mammography DICOM data for model training.

## Objective

A single, clear workflow:

`data/raw/` (+ `data/raw_test/`) -> `create_dataset.py` -> `dataset/processed/` -> `dataset/labels.csv` -> `main.py` -> `evaluate.py`

## Directory Convention

    Medical-AI-Core/
    ├── data/
    │   ├── raw/                    # Train/Val source DICOM files
    │   └── raw_test/               # Test-only source DICOM files
    ├── dataset/
    │   ├── processed/              # PNG outputs generated by create_dataset.py
    │   └── labels.csv              # Prepared by the user for training/evaluation
    ├── result/                     # Model weights and Grad-CAM outputs
    ├── preprocessing/
    ├── training/
    ├── evaluation/
    ├── create_dataset.py
    ├── main.py
    ├── evaluate.py
    └── requirements.txt

## What Do the Modules Do?

- `create_dataset.py`
  - Reads files in `data/raw/`
  - Normalizes the DICOM pixel matrix to 8-bit
  - Inverts the image if it is MONOCHROME1
  - Applies smart crop/pad
  - Saves the result as `.png` under `dataset/processed/`

- `preprocessing/cv_transforms.py`
  - CLAHE
  - Wavelet edge boost
  - Canonical smart crop/pad (`AdvancedImageProcessor.smart_crop_and_pad`)

- `preprocessing/smart_cropper.py`
  - Acts as a wrapper so legacy usage points do not break
  - Keeps the actual crop/pad logic in one place (reduces code duplication)

- `training/dataset_loader.py`
  - Expects `dataset/labels.csv`
  - Required columns: `filename`, `label`, `split`
  - Checks for `train|val|test` as the `split`

- `training/model.py`
  - ResNet50-based binary classifier

- `training/trainer.py`
  - Training loop (forward/backward/optimizer)
  - Reports loss after each epoch
  - Saves the model to disk

- `main.py`
  - Training orchestrator
  - Throws explicit errors in case of missing `labels.csv` or `dataset/processed/`

- `evaluate.py`
  - Calculates accuracy/sensitivity/specificity on the test split
  - Saves Grad-CAM visuals under `result/gradcam/`

## End-to-End Pipeline

### 1) Environment Setup

    pip install -r requirements.txt

### 2) Place Raw Data

Copy train/val source DICOM files into `data/raw/`.

Copy test-only DICOM files into `data/raw_test/`.

### 3) Run Preprocessing

    python create_dataset.py

Preprocessing + single labels.csv generation (raw->train/val, raw_test->test) + train-only augmentation:

    python create_dataset.py --generate-labels --augment-train --overwrite-labels

This command does all of the following in order:
- preprocesses DICOM files into `dataset/processed/`
- creates `dataset/labels.csv` with deterministic `train/val` split from `data/raw/`
- assigns all images from `data/raw_test/` to `split=test`
- infers label from laterality (`L -> 0`, `R -> 1`, fallback to `--default-label`)
- augments only `train` images and appends matching rows to `labels.csv`

Expected output: `dataset/processed/*.png`

### 4) Prepare Labels CSV

Example schema for `dataset/labels.csv`:

    filename,label,split
    scan_0001.png,0,train
    scan_0002.png,1,train
    scan_0003.png,1,val
    scan_0004.png,0,test

You can also use the `dataset/labels.example.csv` file as a ready-made template.

Note: `labels.example.csv` is static and is not auto-updated. The generated file is `dataset/labels.csv`.

### 5) Training

    python main.py

Parameter override example using CLI:

    python main.py --epochs 20 --batch-size 8 --learning-rate 1e-4 --csv dataset/labels.csv --img-dir dataset/processed --save-path result/mammography_resnet_custom.pth

Expected output: `result/mammography_resnet.pth`

### 6) Evaluation

    python evaluate.py

Expected output:
- Final metrics in the terminal
- Visuals under `result/gradcam/`

## Notes

- The old `dicom_processor.py` reference has been removed; the official entry point is `create_dataset.py`.
- `data/raw/` is used instead of the old `dataset/raw_dicom/`.
- `labels.csv` can now be generated automatically via `create_dataset.py --generate-labels`.
